V7 UX-FIX — Mini App (round flags + events + search)

Дата:  12 мая 2026
Файлы: app.py · static/index.html · static/styles.css

Что починено
============

1) Флаги стран стали круглыми, но всё видно
-------------------------------------------
Что было:    квадратные 22×22 с border-radius: 4px
Что стало:   круглые 24×24, border-radius: 50%, флаг загружается в
             разрешении w80 (вместо 24×18) и кадрируется по центру с
             cover. Получается читаемый круг — горизонтальные и
             вертикальные полосы видны, центральные эмблемы тоже.

Файлы:
 ✓ styles.css: .league-flag → 50% + inset shadow + 130% size с
   object-position: center
 ✓ index.html: flagSvg() использует https://flagcdn.com/w80/<cc>.png

2) "Ход матча" пуст → теперь работает из коллектора
----------------------------------------------------
Что было:    события приходили только через IGScore /match/info, и
             когда тот не возвращал event-листы — секция была пуста
             ("Пока нет данных по голам и жёлтым карточкам.")
Что стало:   коллектор детектит изменение счёта между снапшотами и
             пишет goal-события в БД самостоятельно. IGScore API
             используется как первый источник (если возвращает —
             хорошо: с именами игроков), DB-события — fallback.

Новая таблица:
   match_events(match_id, event_id, kind, minute, minute_text,
                team, side, player, detail,
                score_home, score_away, detected_at)

Логика:
 ✓ Первый снапшот матча: backfill — если счёт уже 2-1, создаются 3
   маркера с минутой '—' (присоединились поздно)
 ✓ Каждый следующий снапшот: если счёт вырос — пишется goal-событие
   с актуальной минутой и текущим счётом
 ✓ Несколько голов между снапшотами обрабатываются корректно
 ✓ События удаляются вместе с матчем при cleanup'е

Файлы:
 ✓ app.py: CREATE TABLE match_events, _events_from_db(),
   изменён _upsert_live_matches (детект score-increment),
   match_events_cached() — приоритет API > DB
 ✓ _cleanup_finished_matches удаляет события вместе с матчами

3) Чипсы "Все" и "★ Избр" убраны → поиск
----------------------------------------
Что было:    чипсы [Все] [★ Избр] над списком матчей
Что стало:   поле поиска с иконкой "лупа" и кнопкой ✕ для очистки.
             Ищет по командам / лиге / стране (case-insensitive
             substring) с debounce 120ms.

Технические нюансы:
 ✓ Поиск использует только обновление feed-секции (не топбара)
   — input не теряет фокус между нажатиями
 ✓ state.preset больше не управляет фильтрацией (но функция
   setPreset осталась как no-op для обратной совместимости)
 ✓ Кнопка ✕ появляется только когда есть текст
 ✓ Focus-state: бордер становится зелёным, иконка — зелёной

Файлы:
 ✓ index.html: renderTopbar() рендерит .search-row вместо
   .chips-row; onSearchInput(), clearSearch(), refreshMatchFeedOnly()
 ✓ styles.css: .search-row, .search-input, .search-clear
