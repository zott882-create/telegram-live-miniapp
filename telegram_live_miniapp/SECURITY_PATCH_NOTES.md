# Security production patch

Изменения в этой сборке:

1. `/api/notify/clear` и `/api/notify/remove` определяют пользователя только по проверенному Telegram `init_data`. Поля `chat_id`/`user_id` больше не используются для авторизации.
2. Загрузчик логотипов блокирует приватные, loopback, link-local и зарезервированные IP после DNS-резолва, подключается к проверенному IP напрямую, повторно проверяет каждый редирект и не принимает SVG.
3. Растровые логотипы декодируются Pillow и заново сохраняются в WebP без метаданных. Старые SVG и другие активные форматы больше не раздаются.
4. Встроенный `ThreadingHTTPServer` заменён на production WSGI-сервер Waitress. Команда запуска остаётся прежней: `python app.py`.
5. Docker-контейнер запускается от непривилегированного пользователя.

## Запуск

```bash
pip install -r requirements.txt
python app.py
```
