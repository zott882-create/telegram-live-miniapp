Что внутри
==========
1. Премиум-дизайн в чёрно-зелёных цветах главной страницы
   • Тёмно-зелёный фон (#050a07), акцент #22c55e
   • Группировка матчей по лигам как на igscore.net
     (TOP-страны и TOP-лиги выводятся первыми, не в "разнабой")
   • Премиум-типографика, плотные строки матчей, статистика под счётом

2. Фильтры на статистику
   • Минута матча (диапазон)
   • Удары (общее)
   • Удары в створ (общее)
   • Мимо ворот (общее)
   • Опасные атаки (общее)
   • Атаки (общее)
   • Угловые (общее)
   • Жёлтые/красные карточки (общее)
   • Владение (макс. одной команды)
   • Конкретный счёт (запятыми, например 0-0,1-0,0-1)

3. Карточка матча — три таба
   • Лайв-стат: владение, удары, в створ, мимо, атаки, опасные, угловые, карточки
     (с горизонтальными барами home/away)
   • Все цифры: полная таблица по обеим командам
   • Среднее: средний тотал по 10 матчам + ориентир + 0:0 счётчик

4. Вкладка избранных
   • localStorage (`miniapp-v5-favorites`)
   • Только лайв-матчи из избранного
   • Кнопка ★ на каждой строке + на карточке матча

5. Уведомления
   • Вкладка "Уведомления" в нижнем меню
   • Фильтр уведомлений редактируется через ⚙ → bottom sheet
   • Когда найдётся матч под фильтр:
       - in-app алерт сразу появляется в истории
       - сервер шлёт push в Telegram (если BOT_TOKEN задан)
   • Cooldown 600s на матч (нельзя спамить)

6. Логотипы команд
   • Backend ищет логотипы в homeTeam/awayTeam и похожих полях API
   • Найденные logo URL сохраняются в data/team_logos.sqlite3
   • По умолчанию файлы скачиваются локально в data/team_logos/
   • Frontend показывает логотипы в списке матчей и карточке матча
   • Если логотипа нет — остаются инициалы команды

7. Фильтры по live-статистике IGScore
   • /api/live теперь подтягивает статистику матчей сразу в ленту
   • Фильтры и уведомления работают по ударам, в створ, мимо, атакам,
     опасным атакам, угловым, владению и карточкам
   • Чтобы отключить обогащение ленты статистикой: LIVE_STATS_IN_FEED=0
   • Лимит матчей для обогащения: LIVE_STATS_MAX_MATCHES, default 120
   • Количество параллельных запросов: LIVE_STATS_WORKERS, default 8

Серверные endpoints
===================
GET  /healthz           — статус
GET  /api/live          — список матчей (с фильтрами для пресета)
GET  /api/match?id=...  — детали + 10-матчевая статистика
POST /api/subscribe     — подписка на уведомления (init_data + filter)
POST /api/notify        — пуш сообщения из открытого приложения
GET  /team-logos/<file> — локально сохранённые логотипы

Переменные окружения
====================
HOST                       — default 0.0.0.0
PORT                       — default 8080
DATA_MODE                  — auto | igscore | sqlite | demo
LIVE_DB_PATH               — путь к SQLite от бота
BOT_TOKEN                  — токен бота для Telegram-пушей (без него → in-app only)
NOTIFY_POLL_INTERVAL       — частота проверки фильтров (sec, default 30)
NOTIFY_COOLDOWN_PER_MATCH  — пауза между уведами по одному матчу (sec, default 600)
TEAM_LOGO_DOWNLOAD          — 1 скачивать логотипы локально, 0 хранить только URL
TEAM_LOGO_MAX_BYTES         — лимит размера логотипа, default 1500000
LIVE_STATS_IN_FEED          — 1 подтягивать статистику в /api/live, 0 отключить
LIVE_STATS_MAX_MATCHES      — сколько live-матчей обогащать статистикой, default 120
LIVE_STATS_WORKERS          — параллельные запросы статистики, default 8

Деплой (Render)
===============
1. Загрузить папку telegram_live_miniapp на GitHub
2. Render → New Web Service
   Root Directory:   telegram_live_miniapp
   Build:            (пусто, нет билда)
   Start command:    python app.py
3. Environment Variables:
       PORT             = (Render задаёт сам)
       DATA_MODE        = auto
       BOT_TOKEN        = <токен от @BotFather>
       LIVE_DB_PATH     = /tmp/data/live_history_v1.sqlite3 (если sqlite)
4. После деплоя — открыть https://твой-домен.onrender.com и убедиться,
   что грузится главный экран
5. В @BotFather → /mybots → Bot Settings → Menu Button → Configure
   Menu Button → ввести URL и текст кнопки "Live матчи"

V5.2 PREMIUM COLLECTOR CACHE
===========================
• Добавлен общий background collector.
• Collector обновляет data/live_cache.sqlite3, а пользователи читают /api/live из базы.
• По умолчанию включено:
    COLLECTOR_ENABLED=1
    LIVE_FROM_DB_ONLY=1
    COLLECTOR_INTERVAL=60
    COLLECTOR_MAX_MATCHES=300
    COLLECTOR_WORKERS=2
    COLLECTOR_REQUEST_DELAY=0.35
    DELETE_FINISHED_MATCHES=1
    FINISHED_MATCH_GRACE_SECONDS=120
• Закончившиеся/пропавшие live-матчи удаляются из live_matches и match_stats после grace-паузы.
• /api/match для collector_db также читает статистику из локальной базы и не создаёт дополнительные запросы от пользователей.
