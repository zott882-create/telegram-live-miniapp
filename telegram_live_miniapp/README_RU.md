# Telegram Live Mini App — v5 PREMIUM

Премиум Mini App для Telegram с лайв-матчами в стиле igscore.net.

## Что нового в v5

1. **Премиум дизайн чёрно-зелёный** — главная как igscore.net, не "в разнабой"
2. **Матчи группируются по лигам** — TOP-страны (Англия, Испания, Италия...) и TOP-лиги выводятся первыми
3. **Фильтры на всю статистику** — удары, в створ, опасные, атаки, угловые, владение, счёт, минуты
4. **Карточка матча с 3 табами** — Лайв-стат, Все цифры, Среднее за 10 матчей
5. **Вкладка избранных** — внизу в навигации
6. **Уведомления** — отдельная вкладка с фильтром, пуши в Telegram и in-app
7. **Логотипы команд** — сервер ищет logo-поля в IGScore API, сохраняет URL/локальную копию в SQLite и показывает логотипы в списке и карточке матча

## Запуск локально

```bash
cd telegram_live_miniapp
python3 app.py
```

Открыть: http://127.0.0.1:8080

Можно сразу попробовать demo-режим (3 матча с лайв-статистикой):
```bash
DATA_MODE=demo python3 app.py
```

## Деплой на Render

1. Загрузи папку `telegram_live_miniapp` на GitHub
2. Render → New → Web Service → выбрать репозиторий
3. **Root Directory**: `telegram_live_miniapp`
4. **Start command**: `python app.py` (Render найдёт сам через Procfile)
5. Environment Variables:

| Переменная | Значение | Зачем |
|---|---|---|
| `DATA_MODE` | `auto` | auto / igscore / sqlite / demo |
| `BOT_TOKEN` | `<токен @BotFather>` | для пушей в Telegram |
| `LIVE_DB_PATH` | `/path/to/live_history.sqlite3` | если используешь sqlite-режим |
| `NOTIFY_POLL_INTERVAL` | `30` | частота проверки (сек) |
| `NOTIFY_COOLDOWN_PER_MATCH` | `600` | пауза между пушами по одному матчу |
| `TEAM_LOGO_DOWNLOAD` | `1` | скачивать найденные логотипы локально в `data/team_logos/`; `0` = хранить только URL |
| `TEAM_LOGO_MAX_BYTES` | `1500000` | максимальный размер одного файла логотипа |

6. После деплоя → @BotFather → /mybots → бот → Bot Settings → Menu Button → ввести URL приложения

## Endpoints

| Метод | Путь | Что делает |
|---|---|---|
| GET | `/healthz` | пинг |
| GET | `/api/live` | список лайв-матчей с группировкой по странам/лигам |
| GET | `/api/match?id=...` | детали матча + 10-матчевая статистика |
| POST | `/api/subscribe` | подписка на уведомления (init_data + filter) |
| POST | `/api/notify` | пуш Telegram-сообщения из открытого приложения |
| GET | `/team-logos/<file>` | отдаёт локально сохранённые логотипы команд |

## Как работают логотипы команд

1. При загрузке лайв-матчей сервер проверяет `homeTeam` / `awayTeam` и прямые поля матча (`homeLogo`, `homeLogoUrl`, `homeTeamLogo`, `awayLogo` и похожие).
2. Если логотип найден, он записывается в `data/team_logos.sqlite3`.
3. По умолчанию файл также скачивается в `data/team_logos/`, а frontend получает локальный путь вида `/team-logos/<file>`.
4. Если скачать не получилось, приложение всё равно сохранит внешний URL.
5. Если у команды нет логотипа, frontend показывает инициалы команды, чтобы карточка не ломалась.

## Как работают уведомления

1. Юзер открывает Mini App из бота → JS читает `tg.initData` (подписан ботом)
2. Юзер настраивает фильтр (минуты, удары и т.д.) и включает Toggle
3. Frontend POST-ит `/api/subscribe` с init_data + фильтром
4. Сервер верифицирует HMAC-подпись init_data → сохраняет `{chat_id, filter}` в `data/notify_subs.json`
5. Фоновый поток каждые 30 сек тянет `/api/live` и проверяет все подписки
6. Когда матч проходит фильтр — `sendMessage` в Telegram с кнопкой "↗ IGScore"
7. Cooldown 600s на один матч защищает от спама
8. Параллельно фронт сам добавляет алерт в in-app историю (без задержки)

**Без `BOT_TOKEN`** push в Telegram отключается, но in-app алерты в "Уведомления" работают.

## Совместимость с v4

- Endpoints `/api/live` и `/api/match` остались как были (бэкенд обратно-совместим)
- Структура полей матча та же — добавились опциональные `stats: {...}`, `home_logo`, `away_logo`

## Обновление: фильтры по всей live-статистике

Раньше в режиме IGScore список `/api/live` отдавал пустой `stats: {}` и статистика подгружалась только в карточке матча через `/api/match`. Из-за этого фильтры в общей ленте реально отрабатывали в основном по минуте и счёту.

Теперь backend сразу обогащает live-матчи статистикой IGScore и кладёт её в `match.stats`, поэтому фильтры и уведомления работают по:

- ударам;
- ударам в створ;
- ударам мимо;
- атакам;
- опасным атакам;
- угловым;
- владению;
- жёлтым и красным карточкам.

Новые переменные окружения:

| Переменная | Значение | Зачем |
|---|---|---|
| `LIVE_STATS_IN_FEED` | `1` / `0` | включить или отключить статистику прямо в `/api/live` |
| `LIVE_STATS_MAX_MATCHES` | `120` | максимум live-матчей, для которых backend подтянет статистику |
| `LIVE_STATS_WORKERS` | `8` | сколько параллельных запросов статистики делать |
| `IGSCORE_STAT_TIMEOUT` | `6` | timeout одного запроса статистики, в секундах |

Если матчей очень много и сервер на бесплатном тарифе тормозит, можно поставить `LIVE_STATS_MAX_MATCHES=60` или `LIVE_STATS_IN_FEED=0`.

## Обновление: режим общего collector-кеша для продажи

Эта версия умеет работать по коммерческой схеме: сайт/IGScore опрашивает только один общий серверный collector, а все пользователи Mini App получают уже готовые данные из локальной SQLite-базы.

Схема:

```text
IGScore API
  ↓ один background collector
 data/live_cache.sqlite3
  ↓
/api/live для всех пользователей
```

Так 100, 500 или 1000 пользователей не увеличивают количество запросов к источнику. Фильтры по ударам, атакам, угловым, карточкам, минутам и счёту работают по твоей базе.

Рекомендуемые настройки для обновления до 300 live-матчей примерно раз в минуту:

| Переменная | Рекомендуемое значение | Зачем |
|---|---:|---|
| `COLLECTOR_ENABLED` | `1` | включает общий сборщик |
| `LIVE_FROM_DB_ONLY` | `1` | `/api/live` отдаёт только локальный кеш |
| `COLLECTOR_INTERVAL` | `60` | пауза между циклами обновления |
| `COLLECTOR_MAX_MATCHES` | `300` | максимум матчей, по которым обновлять статистику |
| `COLLECTOR_WORKERS` | `2` | параллельные воркеры статистики |
| `COLLECTOR_REQUEST_DELAY` | `0.35` | задержка между запросами внутри каждого воркера |
| `COLLECTOR_JITTER` | `5` | небольшой случайный разброс, чтобы циклы не были идеально одинаковыми |
| `DELETE_FINISHED_MATCHES` | `1` | удалять матчи, которые пропали из live |
| `FINISHED_MATCH_GRACE_SECONDS` | `120` | подождать 2 минуты перед удалением, чтобы матч не мигал при временном сбое API |

Важно: старые переменные `LIVE_STATS_IN_FEED`, `LIVE_STATS_MAX_MATCHES`, `LIVE_STATS_WORKERS` оставлены для совместимости, но в новом режиме основной поток — это collector + `data/live_cache.sqlite3`.

`/healthz` теперь показывает состояние collector: последний запуск, ошибки, количество матчей и обновлённой статистики.

## Локальные флаги стран

В этой версии флаги стран лежат внутри проекта: `static/flags/*.svg`.
Mini app больше не использует картинки лиг в заголовках, потому что у IGScore они могут выглядеть как непонятные иконки турниров.
Если IGScore отдаёт регион вроде `AFRICA` или `ASIA`, backend пытается определить страну по названию лиги/команд, например `Ethiopian Premier League` -> `Ethiopia` -> `ET`.



## Миграция под много пользователей

В версии V9.41 пользовательские уведомления и найденные матчи хранятся в `data/notify_state.sqlite3` вместо JSON-файлов. Старые `data/notify_subs.json` и `data/notify_matches.json` автоматически мигрируют при первом запуске. Подробности в `MIGRATION_MANY_USERS_RU.md`.
